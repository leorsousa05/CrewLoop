# Output Checklist

Use this as the final quality gate before handing off to Engineer.

## Must pass

- [ ] The design tells a product story.
- [ ] The visual direction is explicit and singular.
- [ ] The output cites the reference files that informed the direction.
- [ ] The color system is semantic and custom, not generic.
- [ ] The typography system feels intentional and not default.
- [ ] The layout has hierarchy and tension.
- [ ] Component states are defined.
- [ ] Real states are defined: loading, empty, error, skeleton, success, offline.
- [ ] Motion has reduced-motion fallback.
- [ ] Accessibility constraints are stated.
- [ ] The result does not read like a template.

## Slop detector

If the design feels interchangeable with any other SaaS or landing page, rework:
- the typography,
- the palette,
- the layout rhythm,
- or the motion language.
