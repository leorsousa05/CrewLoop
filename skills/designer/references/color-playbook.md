# Color Playbook

Color should make the interface feel specific.

## Palette rules

- Choose one dominant color family.
- Choose one accent family for calls to action and important states.
- Use neutrals to support the story, not to hide uncertainty.
- Build semantic tokens around meaning, not appearance.

## Token thinking

- `--bg-primary` is the atmosphere.
- `--bg-surface` is where content sits.
- `--bg-elevated` is for overlays and transient layers.
- `--text-primary` should carry the message.
- `--text-secondary` should support, not compete.
- `--accent` should be the clearest interactive signal.

## What to avoid

- White background plus random purple gradient as the default move.
- Rainbow palettes without a narrative.
- Neon colors with no functional role.
- Equal-weight colors that make the interface feel undecided.

## Good color questions

- What emotion should the product create in the first 3 seconds?
- Which color supports trust?
- Which color is reserved for action?
- Which colors should only appear in small doses?
