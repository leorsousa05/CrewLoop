# Layout Patterns

These are compositional starting points, not implementation recipes.

## Patterns

### Editorial lead
- Large headline block.
- Supporting copy column.
- Secondary details in a smaller, denser rail.
- Best when the first job is to explain, not to demo.

### Asymmetrical hero
- Main message on one side.
- Feature or visual proof on the other.
- Keeps the eye moving and avoids center-stack sameness.

### Bento matrix
- Mix 1x1, 1x2, and 2x2 blocks.
- Use one oversized anchor card and several smaller support cards.
- Great for product index pages and complex SaaS homepages.

### Split comparison
- Left side shows friction or old behavior.
- Right side shows the improved state.
- Useful when the design needs to demonstrate a before/after case.

### Dense utility stack
- Tight vertical rhythm.
- Clear dividers.
- Compact filters, stats, and actions.
- Good for console surfaces and dashboards.

## Composition rules

- Never make every block the same height unless the content truly demands it.
- Let one component dominate the visual rhythm.
- Keep mobile layouts strong by preserving hierarchy, not by flattening everything.
