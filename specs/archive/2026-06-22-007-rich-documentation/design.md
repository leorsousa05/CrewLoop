# Design

## Approach

1. Update `docs/sidebars.js` to define explicit categories and ordering.
2. Create new Markdown pages under `docs/docs/` for the new sections.
3. Rewrite existing skill pages with richer content.
4. Split workflow into multiple pages if needed.
5. Build and validate.

## Rollback

The changes are documentation-only and can be reverted by restoring the previous `docs/docs/` content and `sidebars.js`.
