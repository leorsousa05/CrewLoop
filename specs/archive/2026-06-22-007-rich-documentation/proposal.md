# Proposal: Expand Docusaurus documentation with deep skill guides and richer navigation

## Problem

The current documentation site has only three sections (Getting Started, The Crew, Workflow) and the skill pages are too shallow. The end-to-end workflow is hard to follow, and users cannot understand what each skill is capable of or when to invoke it.

## Why now

A rich documentation site is essential for adoption. Users need deep explanations, concrete examples, clear decision trees, and a logical information architecture.

## Scope

In scope:
- Restructure Docusaurus sidebar into richer sections.
- Rewrite skill pages with deep explanations, responsibilities, examples, anti-patterns, and handoff rules.
- Add new pages: Why CrewLoop, Concepts, Installation, Usage Examples, Decision Trees, Artifacts, Contributing.
- Improve workflow pages with simplified and detailed views.
- Add concrete examples of complete tasks flowing through the crew.

Out of scope:
- Changing skill logic or AGENTS.md rules.
- Adding interactive components.
