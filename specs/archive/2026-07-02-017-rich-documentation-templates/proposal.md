# Proposal: Rich Documentation Templates in Docs-Writer Skill

## Motivation
Current documentation generated by the `docs-writer` skill is too plain and lacks modern visual styling cues found in popular open-source and professional repository documentation (e.g., Ponytail, Caveman, Warp, Catppuccin). Users want documentations to have high-impact centered headers, badges, comparative tables, ASCII cards, callouts, and collapsible sections.

## Scope
- Update `skills/docs-writer/SKILL.md` to incorporate visual rules, guidelines for images, comparative tables, and collapsible elements.
- Update `skills/docs-writer/references/section-templates.md` to include visual skeletons (badges, center branding, tables, metrics, etc.) for CLI, Library, and App project types.
- Update `skills/docs-writer/references/quality-checklist.md` to include checks for visual structure, accessibility/legibility of visual assets, and content formatting.

## Constraints
- Must remain compatible with `python3 scripts/validate-skills.py`.
- No functional code changes. Only Markdown skill and reference updates.
