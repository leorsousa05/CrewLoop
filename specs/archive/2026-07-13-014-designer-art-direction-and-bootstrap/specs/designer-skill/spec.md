# Designer Skill Delta

## Added
- The skill must commit to one clear aesthetic direction before defining tokens or components.
- The skill must explicitly reject generic AI-default patterns, including weakly differentiated SaaS gradients, random glassmorphism, and non-committal typography.
- The skill must cite the relevant reference files from `skills/designer/references/` when making layout, color, motion, and typography choices.
- The skill must produce a case-study-style output with problem, audience, insight, solution, and traceability.

## Modified
- The discovery flow is no longer open-ended. It should resolve the visual direction using the brief and the reference library rather than asking the user for multiple design decisions.
- The output structure becomes stricter and more reusable so future engineers can implement without guessing.

## Removed
- Any wording that makes the skill sound optional or permissive about generic design defaults.
- Any suggestion that a minimal template aesthetic is acceptable by default.
