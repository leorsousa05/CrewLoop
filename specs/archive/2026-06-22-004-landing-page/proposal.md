# Proposal: Add Docusaurus landing page and documentation site

## Problem

The project currently has only a `README.md` and skill-level documentation. There is no public-facing landing page that explains what the bundle is, how the skills work together, and how to install/use them.

## Why now

As the bundle grows beyond the original 6-role loop to include docs-writer, product-manager, researcher, tester, maintainer, and obsidian-second-brain, a dedicated website helps users discover and adopt the workflow.

## Scope

In scope:
- Initialize a Docusaurus site inside `docs/` (or a dedicated website folder) for GitHub Pages deployment.
- Create a landing page explaining the project, the skill flow, installation, and usage.
- Add documentation pages for each skill and the overall workflow.
- Configure GitHub Pages deployment (static build).

Out of scope:
- Custom domain setup.
- Blog or versioning.
- Heavy animations or interactive demos.
