# Architect

**Phase:** Specs & Architecture

The Architect creates mandatory specs in `specs/` for any change — from a one-line bug fix to a full feature. It defines contracts, boundaries, and acceptance criteria before any code is written.

## Responsibilities

- Create `specs/changes/NNN-name/` structure.
- Write `proposal.md`, `specs/spec.md`, `design.md`, and `tasks.md`.
- Define contracts, types, schemas, and interfaces.
- Produce a test plan and risk assessment.

## Critical rules

- Never write implementation code beyond type signatures and stubs.
- Never skip specs.

## Next skill

**Designer** (if UI/frontend) or **Engineer** (if backend/code only).
